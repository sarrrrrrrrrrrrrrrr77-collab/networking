الوحدة 01

مفهوم عنوان الـ IP وفئاته (Classes)

عنوان الـ IP هو "هوية" كل جهاز على الشبكة. يتكوّن من 4 أجزاء (تسمى Octets) مفصولة بنقاط، كل جزء رقم بين 0 و255 (لأن كل Octet هو 8 خانات ثنائية Binary)، مثل: 192.168.1.10

كل Octet في الحقيقة هو رقم ثنائي (Binary) من 8 خانات، ولهذا أقصى قيمة ممكنة له هي 255:

11000000= 192
10101000= 168
00000001= 1
00001010= 10
الفئة النطاق (Range) القناع الافتراضي الاستخدام الشائع
Class A 1.0.0.0 – 126.255.255.255 255.0.0.0 شبكات ضخمة جدًا (دول، مؤسسات كبرى)
Class B 128.0.0.0 – 191.255.255.255 255.255.0.0 شبكات متوسطة (جامعات، شركات كبيرة)
Class C 192.0.0.0 – 223.255.255.255 255.255.255.0 شبكات صغيرة (المنازل، المكاتب الصغيرة)
Class D 224.0.0.0 – 239.255.255.255 — البث المتعدد (Multicast)
Class E 240.0.0.0 – 255.255.255.255 — محجوزة للتجارب والاستخدامات المستقبلية
ملاحظة: العنوان 127.0.0.0 – 127.255.255.255 غير مستخدم لأي جهاز حقيقي على الشبكة، بل محجوز بالكامل لعنوان "Loopback" الذي يشير للجهاز نفسه (مثل 127.0.0.1). لهذا لا تجده ضمن أي Class تشغيلية.
📌 ملاحظة مهندسين

نظام الـ Classes (A, B, C) هو الطريقة التأسيسية القديمة لفهم الشبكات، وتُعرف باسم Classful Addressing. في الواقع العملي اليوم، تم استبدال هذا النظام بالكامل بنظام أحدث وأكثر مرونة يُسمى CIDR — أو النظام غير الفئوي (Classless Inter-Domain Routing) — بموجب المعايير العالمية لمنظمة IETF.

بمعنى آخر: تعلّم الـ Classes هنا هو خطوة تأسيسية لفهم "كيف بدأت الفكرة"، أما الأساس الذي تعمل به الشبكات والراوترات فعليًا اليوم فهو الـ CIDR، وهو ما سنعتمد عليه لاحقًا في وحدات التقسيم الفرعي المتقدّم (VLSM).

الوحدة 02

معنى الـ Subnet Mask

قناع الشبكة (Subnet Mask) هو رقم يُكتب بنفس شكل الـ IP، لكن وظيفته مختلفة تمامًا: هو اللي "يحدد" أي جزء من عنوان الـ IP يمثل الشبكة نفسها، وأي جزء يمثل الجهاز داخلها.

كل فئة من فئات الـ IP لها قناع افتراضي:

  • Class A 255.0.0.0
  • Class B 255.255.0.0
  • Class C 255.255.255.0
فكّر في القناع كأنه "حدود الحي": الجزء المغطى بالقناع (255) يقول "هذا رقم الحي/الشبكة"، والجزء المتبقي (0) يقول "وهذا رقم البيت/الجهاز داخل نفس الحي".

كيف يعمل القناع فعليًا؟ (عملية AND الثنائية)

الجهاز يقارن كل خانة من عنوان الـ IP مع نفس الخانة في القناع باستخدام عملية منطقية تسمى AND: إذا كانت خانة القناع 1 يُحتفظ بالرقم كما هو (جزء الشبكة)، وإذا كانت 0 يتحوّل الناتج لصفر (جزء الجهاز).

11000000IP: 192
10101000IP: 168
00000001IP: 1
00001010IP: 10
11111111قناع: 255
11111111قناع: 255
11111111قناع: 255
00000000قناع: 0
الناتج (Network ID): 192.168.1.0 — والباقي (10) هو رقم الجهاز (Host ID) داخل هذه الشبكة.
💡 كيف يعرف نظام التشغيل جزء الشبكة من جزء الجهاز؟

يقوم المعالج بعملية منطقية تسمى ANDing بين عنوان الـ IP وقناع الشبكة (Subnet Mask) — خانة بخانة، بشكل ثنائي. النتيجة الفورية لهذه العملية هي استخراج الـ Network ID مباشرة، دون الحاجة لأي حسابات معقّدة.

IP AND Subnet Mask = Network ID

ملاحظة: الـ Subnet Mask الافتراضي لكلاس C هو 255.255.255.0، ويعني أن أول 24 بت من العنوان محجوزة تمامًا للشبكة (وهو ما يُكتب اختصارًا بصيغة CIDR كـ /24).

الوحدة 03

الـ Network ID والـ Host ID

بالاعتماد على الـ Subnet Mask، نقدر نقسّم أي عنوان IP إلى جزئين: جزء ثابت يمثل الشبكة (Network ID) وجزء متغيّر يمثل كل جهاز فيها (Host ID).

مثال 1: عنوان 192.168.1.10 بقناع 255.255.255.0 (Class C)

192Network
168Network
1Network
10Host
Network ID — رقم الشبكة نفسها، ثابت لكل الأجهزة المتصلة بها Host ID — رقم الجهاز الفريد داخل نفس الشبكة
بمعنى آخر: كل الأجهزة المتصلة بنفس الراوتر المنزلي غالبًا تشترك في نفس الـ Network ID (مثل 192.168.1)، وتختلف فقط في آخر رقم (Host ID) مثل .10 أو .11 أو .12.

مثال 2: عنوان 10.20.5.8 بقناع 255.0.0.0 (Class A)

10Network
20Host
5Host
8Host
هنا الجزء المخصص للشبكة أصغر (Octet واحد فقط)، بينما الجزء المتاح للأجهزة أكبر بكثير (3 Octets)؛ ولهذا شبكات Class A تتحمّل عدد أجهزة أكبر بكثير من Class C.
الوحدة 04

مقدمة سريعة عن CIDR

بدل ما نكتب القناع كامل بصيغة 255.255.255.0، يمكن اختصاره برمز يوضع بعد شرطة مائلة (Slash) يمثل عدد الخانات الثنائية "المشغولة بالواحدات" في القناع — يسمى هذا الترميز CIDR.

/8255.0.0.0
/16255.255.0.0
/24255.255.255.0
/25255.255.255.128
/26255.255.255.192
/30255.255.255.252
مثال: كتابة 192.168.1.0/24 تعني نفس معنى كتابة الشبكة 192.168.1.0 بقناع 255.255.255.0 — فقط بصيغة مختصرة أسهل في التوثيق وإعدادات الراوترات الحديثة.
الوحدة 05

Public IP مقابل Private IP

العنوان الخاص (Private) يُستخدم داخل شبكتك المحلية فقط، بينما العنوان العام (Public) هو الذي يُعرّف جهازك أو شبكتك أمام الإنترنت بالكامل.

نطاقات الـ Private IP الشائعة

  • صف كبير 10.0.0.0 – 10.255.255.255
  • صف متوسط 172.16.0.0 – 172.31.255.255
  • صف منزلي 192.168.0.0 – 192.168.255.255
هذه العناوين تتكرر في ملايين الشبكات المنزلية حول العالم في نفس اللحظة، لأنها غير مرئية مباشرة على الإنترنت.

مفهوم NAT (بشكل مبسّط)

الراوتر يقوم بعملية تسمى NAT (Network Address Translation): يترجم عناوين أجهزتك الخاصة المتعددة إلى عنوان عام واحد فقط عند خروجها للإنترنت.

لماذا لا نستخدم عناوين عامة لكل الأجهزة مباشرة؟

عدد عناوين IPv4 العامة محدود عالميًا، ولو خُصص عنوان عام مستقل لكل جهاز في العالم لنفدت الأرقام منذ سنوات. بفضل NAT والعناوين الخاصة، يمكن لملايين الأجهزة أن تشترك داخليًا في عناوين متكررة بينما تظهر للإنترنت بعنوان عام واحد فقط لكل شبكة.

🌐 سر تقني

العناوين الخاصة (Private IPs) مثل التي تبدأ بـ 192.168.x.x تُستخدم مجانًا داخل شبكتك المحلية (البيت أو الجامعة) فقط، ولا يمكنها الخروج للإنترنت مباشرة بهذا الشكل.

عندما تتصفح الإنترنت، يقوم الراوتر بتقنية تسمى NAT/PAT (NAT Overload) بتحويل كل هذه العناوين الداخلية المختلفة إلى عنوان عام (Public IP) واحد فقط ومشترك، تخرج به كل أجهزة البيت إلى العالم الخارجي — وليس كما يظن البعض أن كل جهاز يأخذ عنوانًا عامًا مستقلًا به.

هل أحتاج خلفية سابقة في الشبكات؟

لا. هذا الكورس مصمم للمبتدئين تمامًا، ولا يفترض أي معرفة مسبقة سوى استخدام الحاسوب بشكل عام.

كم تستغرق مدة إتمام هذه الوحدات الأربع؟

يمكن إنهاؤها خلال جلسة أو جلستين مركّزتين، فالهدف هو بناء أساس واضح وسريع قبل التوسع لاحقًا.

هل يوجد تطبيق عملي أو تمارين؟

كل وحدة تحتوي على أمثلة رقمية محلولة خطوة بخطوة، وسيتم إضافة تمارين تفاعلية في الإصدارات القادمة من الكورس.

ما الفرق بين هذا الكورس والمستوى المتقدم لاحقًا؟

هذا الكورس سطحي ومكثّف بقصد؛ يغطي المفاهيم الجوهرية فقط. أما Subnetting المتقدم والتوجيه والتبديل فسيكون ضمن مستوى لاحق.
